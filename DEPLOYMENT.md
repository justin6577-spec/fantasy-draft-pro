# Fantasy Draft Pro — deployment & Play Store pipeline

Prepared 2026-07-17. Backend deploys to Render via the `render.yaml` blueprint
in this repo; the mobile app then gets rebuilt against the public API URL.

## Already done (automated)

- `Dockerfile` + `.dockerignore` — builds the API workspace, runs Prisma
  migrations on boot (`migrate deploy`), serves on `$PORT`, health at `/health`.
- `render.yaml` — one-click blueprint: Docker web service + managed Postgres
  (free plan) + managed Redis/Key-Value (free plan). JWT secrets auto-generated
  by Render. Only DATABASE_URL/REDIS_URL/JWT secrets are required by
  `apps/api/src/config/env.ts`; all integrations (Yahoo, OpenAI, RevenueCat,
  social sign-in) are optional and off by default (`NEWS_PROVIDER=mock`).
- Android upload keystore: `apps/mobile/fantasydraft-upload.keystore`
  (gitignored). Credentials live in `~/.gradle/gradle.properties`
  (`FANTASYDRAFT_UPLOAD_*`), outside the repo. Release signing is wired in
  `apps/mobile/android/app/build.gradle`.
  ⚠ `android/` is gitignored and regenerated by `expo prebuild`; if you ever
  run `prebuild --clean`, re-apply the release `signingConfigs` block.

## Human steps (in order)

1. Create an empty GitHub repo (e.g. `fantasy-draft-pro`) — then the local
   repo can be pushed (`git remote add origin ... && git push -u origin main`).
2. Create a Render account (render.com) → New → Blueprint → select the repo.
   Render reads `render.yaml` and provisions everything. Web service plan is
   `starter` (~$7/mo); Postgres/Redis start free.
3. Copy the service URL (e.g. `https://fantasy-draft-api.onrender.com`) —
   verify `<url>/health` returns `{"status":"ok"}`.
4. Rebuild the app against it (automated, see below), create the app in Play
   Console (Create app: "Fantasy Draft Pro", en-US, App, Free), and the AAB
   can be uploaded with the existing Play service account.

## Rebuilding the AAB (automated once the URL exists)

```powershell
cd "apps/mobile"
Set-Content .env "EXPO_PUBLIC_API_URL=https://<your-service>.onrender.com"
cd android; .\gradlew bundleRelease
# output: android/app/build/outputs/bundle/release/app-release.aab
```

Note: free-plan Render Postgres pauses/expires after 30 days and the web
service spins down when idle (first request after idle takes ~30s). Fine for
closed testing; upgrade before public launch.
